Product Requirement Document (PRD)
Product: akash.dev (Personal Portfolio Website)
1. Purpose
To build a minimalistic, modern, and unique portfolio website that:
- Showcases Akash as a Developer, Competitive Programmer, SIH Winner
- Highlights projects 
- Attracts recruiters and internship opportunities
- Reflects clean design and technical depth
- Acts as a personal brand hub
2. Goals & Success Metrics
Primary Goals:
- Present skills clearly within 30 seconds
- Make projects easy to explore
- Provide simple contact access
- Establish professional credibility

Success Metrics:
- Recruiters spend more than 2 minutes on site
- Contact form submissions
- Project GitHub clicks
- Resume downloads
3. Target Users
- Recruiters (Tech companies)
- Startup founders
- Hiring managers
- Fellow developers
- Internship coordinators
4. Design Principles
- Minimalistic (lots of white space)
- Clean typography
- Smooth but subtle animations
- Dark mode by default
- Modern glass / soft shadow UI
- Fast loading (less than 2 seconds)
5. Core Features
Hero Section:
- Name and professional tagline
- Short introduction
- Call-to-action buttons (View Projects, Download Resume)

Skills Section:
- Languages, Backend, AI/ML, Tools
- Minimal skill cards with hover effects

Projects Section:
- Project name
- Short description
- Tech stack
- GitHub link
- Live demo (if available)

Resume Section:
- One-click download
- View resume in browser

Contact Section:
- Email
- LinkedIn
- GitHub
- Simple contact form
6. Unique Differentiation Ideas
- Subtle animated background grid
- Command palette (like VS Code)
- Terminal-style intro
- Smooth scroll snapping
- Minimal easter egg
7. Non-Functional Requirements
Performance:
- Lighthouse score above 90
- Lazy load images
- Optimize fonts

Accessibility:
- Proper contrast
- Keyboard navigation
- Semantic HTML

Responsiveness:
- Fully mobile responsive
- Tablet optimized

 ├── Projects
 ├── Resume
 └── Contact
